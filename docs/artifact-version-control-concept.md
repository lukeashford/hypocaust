# Artifact Version Control - Final Concept

This document describes the simplified artifact version control system for Hypocaust. It replaces
the overly complex branch-based system with a streamlined commit graph approach.

---

## Core Philosophy

1. **Stateless backend**: No concept of "checkout". The backend knows the full commit graph; the
   frontend tells it which commit to start from.
2. **Name-based artifacts**: Artifacts have semantic file names (e.g., `protagonists_dog`, `script`)
   instead of UUIDs for LLM-friendly references.
3. **Commit-time persistence**: Operators don't save artifacts during execution. They record
   intentions in a diff; the commit process downloads and stores files.
4. **Everything is a task**: All operations (create, edit, delete) flow through the
   DecomposingOperator. No direct REST manipulation of artifacts.
5. **Lean implementation**: Projects usually have ~10 artifacts. No need for semantic embeddings.

## Data Model

### Project

Unchanged. A project is a container for runs, commits, and artifacts.

### Run

A run belongs to a project and executes a task.

```java
// RunEntity.java changes
@Entity
class RunEntity extends BaseEntity {

  UUID projectId;           // Existing
  String task;              // Existing
  Status status;            // Existing: QUEUED, RUNNING, COMPLETED, FAILED
  String reason;            // Existing
  Instant startedAt;        // Existing
  Instant completedAt;      // Existing

  // NEW FIELDS:
  UUID baseCommitId;        // The commit this run started from (null for first run)
  UUID commitId;            // The commit produced by this run (null until committed)
}
```

**Key changes**:

- `baseCommitId` tracks where the run started from (which commit's state)
- `commitId` is set atomically with status=COMPLETED (transactional requirement)
- A run with `commitId != null` MUST have `status == COMPLETED` (invariant)

### Commit

A commit is an immutable snapshot. No branches - just a directed acyclic graph of commits.

```java
record Commit(
    UUID id,
    UUID projectId,
    UUID parentCommitId,  // Previous commit (null for first)
    UUID runId,           // The run that produced this
    String task,          // Verbatim task from user
    String message,       // LLM-generated summary
    Instant timestamp,
    CommitDelta delta
)
```

**Key change**:

- Removed `branchId` - commits form a DAG without named branches
- Added `message` - auto-generated by a small LLM call from the task
- Added `projectId` for direct lookup

### CommitDelta

Records what changed in a commit.

```java
record CommitDelta(
    List<ArtifactChange> added,     // New artifacts
    List<ArtifactChange> edited,    // New versions of existing artifacts
    List<String> deleted            // Artifact names marked as deleted
)

record ArtifactChange(
    String name,          // e.g., "protagonists_dog"

int version           // e.g., 1, 2, 3
)
```

**Key change**: Uses artifact names, not UUIDs. `edited` replaces `updated` with clearer semantics.

### Artifact

```java
record Artifact(
    UUID id,
    UUID projectId,
    UUID commitId,        // The commit that created this version

    // Identity
    String name,          // Semantic file name: "protagonists_dog", "script"
    int version,          // Version counter within this name

    // Content
    Kind kind,            // IMAGE, STRUCTURED_JSON, PDF, AUDIO, VIDEO
    String storageKey,    // For file-based artifacts
    JsonNode content,     // For inline content

    // Metadata
    String description,   // Full description of what's in the artifact
    String prompt,        // The prompt used to generate it
    String model,         // The model used
    JsonNode metadata,    // Additional technical metadata

    // Status
    boolean deleted,      // Soft-delete flag
    Status status         // SCHEDULED, CREATED, CANCELLED
)
```

**Key changes**:

- `name` + `version` replaces semantic anchors with UUID-based identity
- Added `description`, `prompt`, `model` for full provenance
- Added `deleted` flag for soft deletes
- Removed: `anchorDescription`, `anchorRole`, `anchorTags`, `supersededById`, `branchId`,
  `derivedFrom`

## Services

### ArtifactVersionManagementService

Central service for all version control operations.

```java

@Service
class ArtifactVersionManagementService {

  // === Commit Operations ===

  /**
   * Create a commit from the current run's pending changes.
   * Called by TaskService at run completion.
   *
   * @param runId The run to commit
   * @param task The original task (for commit message generation)
   * @param pending The accumulated changes
   * @return The created commit, or empty if no changes
   */
  @Transactional
  Optional<Commit> commit(UUID runId, String task, PendingChanges pending);

  /**
   * Generate a commit message from a task using a small LLM.
   */
  String generateCommitMessage(String task);

  // === Artifact Resolution ===

  /**
   * Get the current artifacts for a commit by traversing history.
   * Walks from the commit back through predecessors, progressively building state.
   */
  List<Artifact> getArtifactsAtCommit(UUID commitId);

  /**
   * Get artifacts for a run.
   * If run has a commit: return artifacts at that commit.
   * If run has no commit yet: return artifacts at baseCommit adjusted by pending changes.
   */
  List<Artifact> getArtifactsForRun(UUID runId);

  /**
   * Get the most recent commit for a project.
   */
  Optional<Commit> getMostRecentCommit(UUID projectId);

  /**
   * Get full commit history for a project.
   */
  List<Commit> getCommitHistory(UUID projectId);

  /**
   * Get all versions of an artifact by name.
   */
  List<Artifact> getVersionHistory(UUID projectId, String artifactName);

  // === Used by commit process ===

  /**
   * Download and store a pending artifact.
   * Called during commit for artifacts that have external URLs.
   * Needs to land in MinIO before the commit is created.
   */
  void materializeArtifact(PendingArtifact pending, UUID commitId);
}
```

### RunContext (Enhanced)

Replaces both `RunContextHolder` and `ExecutionContext`. Thread-local context for the current run.

```java
class RunContext {

  UUID projectId;
  UUID runId;
  UUID baseCommitId;           // Commit this run started from
  PendingChanges pending;      // Accumulated changes

  // === Artifact Hooks (called by operators) ===

  /**
   * Add a new artifact.
   * @throws ArtifactExistsException if name already exists in baseCommit's state
   */
  void addArtifact(PendingArtifact artifact);

  /**
   * Edit an existing artifact (creates new version).
   * @throws ArtifactNotFoundException if name doesn't exist in baseCommit's state
   */
  void editArtifact(String name, PendingArtifact newVersion);

  /**
   * Delete an artifact (soft delete).
   * @throws ArtifactNotFoundException if name doesn't exist in baseCommit's state
   */
  void deleteArtifact(String name);

  /**
   * Check if an artifact name exists in current state.
   */
  boolean artifactExists(String name);

  /**
   * Get current artifact state (base commit, adjusted in this method by pending).
   */
  List<ArtifactSnapshot> getCurrentArtifacts();
}

record PendingArtifact(
    String name,
    Kind kind,
    String description,
    String prompt,
    String model,
    String externalUrl,      // URL to download from (for images)
    JsonNode inlineContent,  // For structured content
    JsonNode metadata
)

record ArtifactSnapshot(
    String name,

int version,
Kind kind,
String description,
boolean isPending        // True if from pending changes, not yet committed
)
```

### RunContextHolder

Simplified thread-local holder.

```java
class RunContextHolder {

  private static final ThreadLocal<RunContext> context = new ThreadLocal<>();

  static void setContext(RunContext ctx);

  static RunContext getContext();

  static void clear();

  // Convenience methods
  static UUID getProjectId();

  static UUID getRunId();

  static void addArtifact(PendingArtifact artifact);

  static void editArtifact(String name, PendingArtifact newVersion);

  static void deleteArtifact(String name);
}
```

### PendingChanges

Accumulator for run changes, used by RunContext. This is materialized as a commit delta.

```java
class PendingChanges {

  List<PendingArtifact> added;
  Map<String, PendingArtifact> edited;  // name -> new version
  Set<String> deleted;                   // names to delete
}
```

## Tools

### ProjectContextTool

A **tool** (not an operator) that provides project context to operators. Answers questions in
natural language.

```java

@Component
class ProjectContextTool implements Tool {

  private final ArtifactVersionManagementService versionService;
  private final ChatClient chatClient;  // Small model for interpretation

  /**
   * Answer a question about project artifacts or version history.
   *
   * Examples:
   * - "What is this project about?"
   * - "What is the artifact name for the picture of our protagonist wearing a suit?"
   * - "What prompt was used for the forest_background artifact?"
   * - "List all current artifacts"
   * - "Show me the version history of hero_image"
   */
  String ask(String question) {
    // 1. Get current run context
    var ctx = RunContextHolder.getContext();

    // 2. Gather relevant data based on question
    var artifacts = versionService.getArtifactsForRun(ctx.runId());
    var history = versionService.getCommitHistory(ctx.projectId());

    // 3. Build context for LLM
    var prompt = buildPrompt(question, artifacts, history);

    // 4. Call small LLM to interpret and answer
    return chatClient.prompt()
        .system("You answer questions about a creative project's artifacts...")
        .user(prompt)
        .call()
        .content();
  }
}
```

**Usage**: Operators call `projectContextTool.ask("What is the artifact name for the dog picture?")`
to resolve natural language references to artifact names and understand their provenance.

## Operators

### ImageGenerationOperator (Adjusted)

Creates new images. Calls `RunContext.addArtifact()`.

```java

@Override
protected OperatorResult doExecute(Map<String, Object> inputs) {
  var prompt = (String) inputs.get("prompt");
  var artifactName = (String) inputs.get("artifactName");  // Required
  var description = (String) inputs.get("description");    // Required

  // Generate image
  var imageUrl = generateImage(prompt);

  // Record in pending changes (NOT saved yet)
  RunContextHolder.addArtifact(PendingArtifact.builder()
      .name(artifactName)
      .kind(Kind.IMAGE)
      .description(description)
      .prompt(prompt)
      .model("dall-e-3")
      .externalUrl(imageUrl)
      .build());

  return OperatorResult.success(...);
}
```

**Key change**: Operator doesn't call `ArtifactService.schedule()` or `updateArtifact()`. It just
records the pending artifact. The commit process downloads and stores it.

### ImageEditOperator (New)

Edits existing images using inpainting/img2img. Calls `RunContext.editArtifact()`.

```java

@Component
class ImageEditOperator extends BaseOperator {

  private final ProjectContextTool projectContext;
  private final ReplicateClient replicateClient;  // For nano banana

  @Override
  protected OperatorResult doExecute(Map<String, Object> inputs) {
    var task = (String) inputs.get("task");  // e.g., "Make the dog black"

    // 1. Resolve which artifact to edit
    var artifactName = projectContext.ask(
        "What is the artifact name for: " + task + "? Reply with just the name.");

    // 2. Get the current artifact
    var artifact = projectContext.getArtifactByName(artifactName)
        .orElseThrow(() -> new ArtifactNotFoundException(artifactName));

    // 3. Edit the image
    var newImageUrl = replicateClient.editImage(
        artifact.storageKey(),
        task,
        artifact.prompt()  // Use original prompt for context
    );

    // 4. Record as edit (new version)
    RunContextHolder.editArtifact(artifactName, PendingArtifact.builder()
        .name(artifactName)
        .kind(Kind.IMAGE)
        .description(artifact.description())  // Preserve description
        .prompt(task)  // New prompt is the edit instruction
        .model("nano-banana")
        .externalUrl(newImageUrl)
        .build());

    return OperatorResult.success(...);
  }
}
```

## Task Progress System

### TaskProgressService

Manages hierarchical task progress for UI display.

```java

@Service
class TaskProgressService {

  private final Map<UUID, TaskTree> trees = new ConcurrentHashMap<>();
  private final EventService eventService;

  /**
   * Publish subtasks for a path.
   * Called by operators to declare their planned work.
   */
  void publishSubtasks(UUID runId, String pathPrefix, List<TaskItem> subtasks) {
    var tree = trees.computeIfAbsent(runId, id -> new TaskTree(id));
    tree.setSubtasks(pathPrefix, subtasks);
    eventService.publish(new TaskProgressUpdatedEvent(runId));
  }

  /**
   * Update a task's status.
   */
  void updateStatus(UUID runId, String taskId, TaskStatus status) {
    var tree = trees.get(runId);
    if (tree != null) {
      tree.updateStatus(taskId, status);
      eventService.publish(new TaskProgressUpdatedEvent(runId));
    }
  }

  /**
   * Get the full task tree for a run.
   */
  Optional<TaskTree> getTree(UUID runId) {
    return Optional.ofNullable(trees.get(runId));
  }
}

record TaskItem(
    String id,           // Hierarchical: "0", "0.1", "0.1.2"
    String description,
    TaskStatus status    // PENDING, IN_PROGRESS, COMPLETED, FAILED, CANCELLED
)

enum TaskStatus {
  PENDING, IN_PROGRESS, COMPLETED, FAILED, CANCELLED
}
```

### Integration with Operators

Operators receive their task path and publish subtasks.

```java
// In DecomposingOperator
@Override
protected OperatorResult doExecute(Map<String, Object> inputs) {
  var task = (String) inputs.get("task");
  var taskPath = RunContextHolder.getContext().getTaskPath();  // e.g., "0.1"

  // After decomposition, publish subtasks
  var subtasks = List.of(
      new TaskItem(taskPath + ".0", "Generate hero image", PENDING),
      new TaskItem(taskPath + ".1", "Create background", PENDING)
  );
  taskProgressService.publishSubtasks(runId, taskPath, subtasks);

  // Execute children, updating status as they progress
  for (int i = 0; i < children.size(); i++) {
    taskProgressService.updateStatus(runId, taskPath + "." + i, IN_PROGRESS);
    // ... execute child ...
    taskProgressService.updateStatus(runId, taskPath + "." + i, COMPLETED);
  }
}
```

## Task Execution Flow

```
1. POST /tasks { projectId, commitId?, task }
   └─ Validate projectId exists
   └─ Resolve baseCommitId (provided or most recent)
   └─ Create Run (status=QUEUED, baseCommitId set)
   └─ Emit RUN_CREATED event
   └─ Submit async execution

2. EXECUTION (async TaskService.executeTask)
   └─ Set RunContext (projectId, runId, baseCommitId, empty pending)
   └─ Emit RUN_STARTED
   └─ DecomposingOperator.execute(task)
   │   └─ Operators call RunContext hooks (addArtifact, editArtifact, deleteArtifact)
   │   └─ Changes accumulate in pending
   └─ On success: commit(), discard pending and cancelled
   └─ On failure: discard all

3. COMMIT (transactional)
   └─ If pending.hasChanges():
   │   └─ Generate commit message from task
   │   └─ For each pending artifact:
   │   │   └─ Download external URL (if any)
   │   │   └─ Store to MinIO
   │   │   └─ Create ArtifactEntity
   │   └─ Create CommitEntity with delta
   │   └─ Update Run (commitId set, status=COMPLETED)
   └─ Else:
       └─ Update Run (status=COMPLETED, no commit)
   └─ Emit RUN_COMPLETED
```

## Artifact Resolution Algorithm

To get artifacts at a given commit:

```java
List<Artifact> getArtifactsAtCommit(UUID commitId) {
  // 1. Build commit chain from root to target
  var chain = new ArrayList<Commit>();
  var current = commitId;
  while (current != null) {
    var commit = commitRepository.findById(current).orElseThrow();
    chain.add(0, commit);  // Prepend to get oldest first
    current = commit.parentCommitId();
  }

  // 2. Replay deltas to build current state
  var state = new HashMap<String, Artifact>();  // name -> latest artifact

  for (var commit : chain) {
    var delta = commit.delta();

    // Add new artifacts
    for (var added : delta.added()) {
      var artifact = artifactRepository.findByCommitIdAndNameAndVersion(
          commit.id(), added.name(), added.version());
      state.put(added.name(), artifact);
    }

    // Apply edits (new versions)
    for (var edited : delta.edited()) {
      var artifact = artifactRepository.findByCommitIdAndNameAndVersion(
          commit.id(), edited.name(), edited.version());
      state.put(edited.name(), artifact);
    }

    // Remove deleted
    for (var deleted : delta.deleted()) {
      state.remove(deleted);
    }
  }

  return new ArrayList<>(state.values());
}
```

## Artifact Reference Syntax

The current implementation uses `@anchor:description` to resolve artifacts by semantic description.
This changes to name-based references.

### Current (to be removed)

```
@anchor:woman in red dress at cafe  →  semantic search  →  ArtifactNode
```

### New (to implement)

```
@artifact:hero_image  →  direct name lookup  →  Artifact
```

Or simply use the artifact name directly in inputs without special syntax, since names are now
explicit:

```json
{
  "values": {
    "targetArtifact": "hero_image",
    "editTask": "Make the hero's hair blonde"
  }
}
```

The `InvokeTool` should be simplified to just resolve artifact names from the current state (base
commit + pending changes) via `RunContext.getArtifactByName(name)`.

## Open Design Decisions

### Artifact Name Generation

Who generates artifact names?

- **Option A**: Operator receives name as input from decomposer
- **Option B**: Operator generates name from description using small LLM

Recommendation: **Option A** - The decomposer knows the project context and can assign meaningful
names. Operators just use them.

### Version Number Strategy

When editing:

- Current artifact is `hero_image` v2
- Edit creates `hero_image` v3

The version is incremented automatically by `RunContext.editArtifact()` based on the highest
existing version for that name.

### Conflict Handling

What if two concurrent runs both add an artifact with the same name?

- The first to commit wins
- The second fails at commit time with a conflict error
- The run is marked FAILED, user must retry with different name

This is acceptable given the expected low concurrency.
